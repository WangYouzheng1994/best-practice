## 1. 适用范围

本文适用于团队内所有服务器部署场景，包括但不限于：

- 数据库服务：MySQL、PostgreSQL、MongoDB、ClickHouse、达梦、人大金仓等；
- 缓存服务：Redis、Memcached 等；
- 消息队列：RabbitMQ、Kafka、RocketMQ、Pulsar 等；
- 搜索与日志：Elasticsearch、OpenSearch、Logstash、Kibana 等；
- 注册配置中心：Nacos、Consul、Etcd、ZooKeeper、Eureka 等；
- 网关与 Web 服务：Nginx、OpenResty、Ingress 网关、静态资源服务等；
- 应用服务：Java、Go、Node.js、Python、.NET 等业务应用；
- 运维监控：Prometheus、Grafana、Alertmanager、Exporter、日志采集 Agent 等；
- 管理平台：CI/CD、制品仓库、堡垒机、任务调度、低代码平台等。

本文只定义团队内部端口规划。具体服务是否允许访问、允许哪些来源访问，应以对应部署手册、安全组、防火墙和 CMDB 记录为准。

## 2. 总体原则

1. **禁止生产环境直接使用常见默认端口。** 例如 MySQL 不使用 `3306`，Redis 不使用 `6379`，PostgreSQL 不使用 `5432`。
2. **同一类服务使用固定端口段。** 端口段内再按实例、角色或集群编号分配具体端口。
3. **端口段应预留扩展空间。** 单一服务类型通常预留 `100` 个端口；集群类服务按服务端口、管理端口、内部通信端口分别预留。
4. **端口用途必须可识别。** 端口号应尽量保留原服务默认端口特征，例如 MySQL 使用 `13306`，Redis 使用 `16379`。
5. **业务应用和中间件分段管理。** 不允许业务应用随意占用数据库、缓存、消息队列、监控等基础服务端口段。
6. **外部访问和内部通信分离。** 对外服务端口、管理端口、集群内部通信端口应分配到不同端口段。
7. **端口必须登记。** 任何生产端口启用前必须记录在资产台账或 CMDB 中，包括服务名、服务器、监听地址、访问来源、责任人和变更记录。
8. **最小开放原则。** 端口不得默认对 `0.0.0.0/0` 开放，必须按业务来源 IP、网段或安全组精确放行。
9. **一机多实例必须显式规划。** 同一服务器部署多个实例时，端口、数据目录、日志目录、systemd 服务名和监控项必须一一对应。
10. **端口变更必须同步文档、配置、监控和访问策略。** 禁止只改应用连接串或只改服务配置。

## 3. 端口分配规则

### 3.1 端口段选择

生产服务优先使用 `10000 ~ 59999` 范围内端口。

| 范围 | 使用要求 |
|---:|---|
| `1 ~ 1023` | 系统知名端口，业务服务禁止占用 |
| `1024 ~ 9999` | 保留给系统组件、临时兼容或历史服务，新服务不推荐使用 |
| `10000 ~ 59999` | 团队标准服务端口段，新增服务优先使用 |
| `60000 ~ 65535` | 保留给临时测试、压测、应急排障，不作为长期生产端口 |

### 3.2 单体实例端口规则

单体实例优先使用服务端口段内的标准端口。例如：

| 服务 | 端口段 | 单体标准端口 |
|---|---:|---:|
| MySQL | `13300 ~ 13399` | `13306` |
| Redis | `16300 ~ 16399` | `16379` |
| PostgreSQL | `15400 ~ 15499` | `15432` |
| MongoDB | `17000 ~ 17099` | `17017` |

### 3.3 多实例端口规则

同一主机部署多个同类实例时，应在同一端口段内连续分配，并同步目录命名。

以 MySQL 为例：

| 实例 | 端口 | 建议实例目录 |
|---|---:|---|
| mysql-01 | `13306` | `/data/mysql/13306` |
| mysql-02 | `13307` | `/data/mysql/13307` |
| mysql-03 | `13308` | `/data/mysql/13308` |

以 Redis 为例：

| 实例 | 端口 | 建议实例目录 |
|---|---:|---|
| redis-01 | `16379` | `/data/redis/16379` |
| redis-02 | `16380` | `/data/redis/16380` |
| redis-03 | `16381` | `/data/redis/16381` |

### 3.4 集群内部通信端口规则

集群类服务必须区分客户端访问端口和节点内部通信端口。

| 服务 | 客户端端口段 | 内部通信端口段 | 说明 |
|---|---:|---:|---|
| Redis Cluster | `16300 ~ 16399` | `36300 ~ 36399` | `cluster-port` 必须显式指定，避免和 Sentinel 端口段混用 |
| Kafka | `29090 ~ 29099` | `29100 ~ 29199` | 客户端监听与 controller/inter-broker 监听分离 |
| Elasticsearch | `19200 ~ 19299` | `19300 ~ 19399` | HTTP 与 transport 分离 |
| ZooKeeper | `12180 ~ 12199` | `12280 ~ 12299` | client、peer、leader election 分离 |
| Etcd | `12370 ~ 12379` | `12380 ~ 12389` | client 与 peer 分离 |

## 6. 端口申请与登记要求

生产端口启用前必须登记以下信息：

| 字段 | 要求 |
|---|---|
| 服务名称 | 系统名、模块名、组件名 |
| 服务类型 | 数据库、缓存、消息队列、应用、监控等 |
| 环境 | 开发、测试、预生产、生产 |
| 主机信息 | 主机名、内网 IP、所属业务 |
| 监听地址 | `127.0.0.1`、内网 IP 或负载均衡地址 |
| 监听端口 | 具体端口号 |
| 访问来源 | 允许访问的 IP、网段、安全组或应用 |
| 责任人 | 业务负责人、运维负责人 |
| 变更记录 | 创建、变更、下线时间与工单编号 |

未登记端口不得上线。临时排障端口使用后必须关闭并补充记录。

## 7. 防火墙与安全组要求

1. 数据库、缓存、消息队列、注册中心、监控管理端口不得对公网开放。
2. 生产端口必须按来源 IP、来源网段或安全组白名单放行。
3. 管理后台、监控平台、CI/CD、制品仓库应优先通过 VPN、堡垒机或内网访问。
4. 对外服务建议由负载均衡、API 网关或 WAF 暴露 `80/443`，后端服务器使用本规范端口。
5. 同一服务涉及多个端口时，必须分别说明用途，不允许用“开放全部端口段”替代精确放行。

主机防火墙基线由《CentOS7.9生产环境初始化手册.md》统一规定，本文不重复维护 `firewalld` 放行规则。访问控制统一由外层防火墙、安全组、ACL、VPN、堡垒机或统一网络策略承载。

禁止示例：

```text
0.0.0.0/0 -> 13306
0.0.0.0/0 -> 16379
0.0.0.0/0 -> 26379
0.0.0.0/0 -> 28080
```

## 8. 部署文档引用要求

所有新增部署文档必须在端口配置章节引用本文档，并明确：

- 所属服务端口段；
- 本文档示例端口；
- 是否禁止默认端口；
- 防火墙或安全组放行范围；
- 端口变更时需要同步修改的位置。

推荐写法：

```text
端口规划应遵循《服务器端口统一规划规范.md》。本文示例端口仅用于标准化部署，实际生产环境必须在 CMDB 或资产台账中登记后使用。
```

## 9. 端口冲突排查命令

检查端口监听：

```bash
ss -lntp
ss -lntp | grep ':13306'
ss -lntp | grep ':16379'
```

检查进程占用：

```bash
lsof -iTCP:13306 -sTCP:LISTEN
lsof -iTCP:16379 -sTCP:LISTEN
```

检查防火墙状态。本文默认 `firewalld` 已按《CentOS7.9生产环境初始化手册.md》统一关闭：

```bash
systemctl is-active firewalld 2>/dev/null || true
systemctl is-enabled firewalld 2>/dev/null || true
iptables -S 2>/dev/null | head -20
```

检查连接连通性：

```bash
nc -vz 10.10.10.10 13306
nc -vz 10.10.10.10 16379
```

## 10. 变更与回滚要求

端口变更必须至少同步以下内容：

1. 服务配置文件；
2. systemd 启动参数；
3. 应用连接串；
4. 防火墙规则；
5. 云安全组；
6. 负载均衡后端配置；
7. 监控采集配置；
8. 告警规则；
9. CMDB 或资产台账；
10. 对应部署手册。

回滚前必须确认旧端口未被其他服务占用，并提前准备：

- 原服务配置文件备份；
- 原防火墙和安全组规则；
- 应用连接串回滚方案；
- 监控与告警恢复方案；
- 业务验证清单。
